UAS Penambangan Data#

Klasifikasi Nilai Akhir Mahasiswa (GRADE) Menggunakan Decision Tree dan Random Forest#

Tools: KNIME Analytics Platform

1. Pendahuluan#

Notebook ini membahas proses analisis penambangan data (data mining) terhadap dataset nilai akademik mahasiswa. Tujuan analisis adalah membangun model klasifikasi yang dapat memprediksi nilai akhir (GRADE) mahasiswa berdasarkan 30 fitur jawaban kuesioner dan kode mata kuliah yang diambil, menggunakan dua algoritma klasifikasi: Decision Tree dan Random Forest.

Seluruh proses pemodelan dilakukan menggunakan KNIME Analytics Platform.

2. Deskripsi Dataset#

Atribut

Keterangan

Jumlah baris (instance)

145 data mahasiswa

Jumlah kolom (atribut)

33 kolom

Missing value

Tidak ada (0 missing value)

Format file

CSV dengan delimiter titik koma ( ; )

Atribut

Tipe Data

Rentang Nilai

Catatan: Dataset tidak disertai data dictionary yang menjelaskan makna spesifik tiap kolom 1-30 (mis. kolom mana mewakili pertanyaan apa). Berdasarkan struktur datanya, kolom-kolom tersebut bersifat ordinal hasil kuesioner.

3. Metodologi: Workflow KNIME#

Proses pemodelan dibangun menggunakan rangkaian node pada KNIME Analytics Platform, mengikuti alur:

Node 1 - CSV Reader#

Membaca file dataset (DATA (1).csv) ke dalam tabel KNIME. Delimiter diatur ke titik koma (;) sesuai format file sumber. Hasil pembacaan menghasilkan tabel berukuran 145 baris × 33 kolom.

Node 2 - Missing Value#

Node Missing Value digunakan untuk memeriksa dan menangani data yang kosong (missing value). Setelah dilakukan pengecekan, dataset ini tidak memiliki missing value sehingga tidak diperlukan proses pengisian atau penghapusan data.

Sebagai antisipasi, KNIME menyediakan beberapa metode penanganan missing value, yaitu:

  • Mean -> mengisi nilai kosong pada kolom numerik dengan nilai rata-rata kolom.

  • Most Frequent Value -> mengisi nilai kosong pada kolom kategorikal dengan nilai yang paling sering muncul (modus).

Pada dataset ini kedua metode tersebut tidak diterapkan karena seluruh data sudah lengkap, namun node tetap digunakan untuk memastikan kualitas data sebelum proses pemodelan.

Node 3 - Column Filter#

Menghapus kolom STUDENT ID dari proses pemodelan karena bersifat identitas unik per mahasiswa dan tidak relevan sebagai fitur prediksi. Kolom yang tersisa setelah node ini adalah fitur 1-30, COURSE ID, dan GRADE.

Node 4 - Number to String#

Mengonversi tipe data kolom GRADE dari Integer menjadi String. Konversi ini diperlukan karena KNIME memerlukan tipe data nominal (String) pada kolom target agar dikenali sebagai kelas klasifikasi, bukan nilai numerik kontinu.

Node 5 - Table Partitioner#

Membagi dataset menjadi data latih (training set, 80%) dan data uji (test set, 20%) secara acak. Data latih digunakan untuk melatih model, sedangkan data uji digunakan untuk menguji performa model pada data yang belum pernah dilihat sebelumnya.

Node 6 - Decision Tree Learner#

Node Decision Tree Learner digunakan untuk melatih model Decision Tree menggunakan data latih dengan GRADE sebagai target klasifikasi. Hasil dari node ini adalah model pohon keputusan yang akan digunakan untuk melakukan prediksi.

Pada proses pembentukan pohon digunakan kriteria Gain Ratio, yaitu metode yang memilih atribut terbaik untuk percabangan dengan mempertimbangkan kualitas pemisahan data serta jumlah nilai unik pada atribut. Metode ini dipilih agar pemilihan atribut lebih seimbang dan tidak bias terhadap atribut yang memiliki banyak kategori.

Node 7 - Decision Tree Predictor#

Menerapkan model Decision Tree yang telah dilatih pada Node 7 untuk memprediksi nilai GRADE pada data uji. Output node ini berupa tabel data uji dengan tambahan kolom hasil prediksi.

Node 8 - Scorer (JavaScript) - Decision Tree#

Membandingkan nilai GRADE aktual dengan hasil prediksi dari Decision Tree Predictor, lalu menghasilkan confusion matrix beserta metrik evaluasi seperti Accuracy, Precision, Recall, F-measure, dan Cohen’s Kappa. Hasil evaluasi node ini menunjukkan Overall Accuracy sebesar 0.231 (23.1%) dengan Cohen’s Kappa 0.086.

Node 9 - Random Forest Learner#

Pada bagian Attribute Selection, kolom GRADE ditetapkan sebagai Target column (kelas yang akan diprediksi). Untuk Training attributes, dipilih opsi Use column, yang berarti model dilatih menggunakan kolom-kolom fitur asli pada dataset (fitur 1-30 dan COURSE ID) secara langsung, berbeda dengan opsi Use fingerprint yang digunakan khusus untuk data bertipe fingerprint/bit vector (umumnya pada data kimia atau molekul).

Node 10 - Random Forest Predictor#

Menerapkan model Random Forest yang telah dilatih untuk memprediksi nilai GRADE pada data uji, dengan cara serupa Decision Tree Predictor.

Node 11 - Scorer (JavaScript) - Random Forest#

Membandingkan nilai GRADE aktual dengan hasil prediksi dari Random Forest Predictor, lalu menghasilkan confusion matrix beserta metrik evaluasi seperti Accuracy, Precision, Recall, F-measure, dan Cohen’s Kappa. Hasil evaluasi node ini menunjukkan Overall Accuracy sebesar 0.345 (34.5%) dengan Cohen’s Kappa 0.206.

Berdasarkan hasil evaluasi, model Random Forest menghasilkan akurasi yang lebih tinggi (34.5%) dibandingkan Decision Tree (23.1%). Hal ini sesuai dengan karakteristik umum kedua algoritma, di mana Random Forest membangun banyak pohon keputusan secara ensemble sehingga lebih tahan terhadap overfitting dan noise dibandingkan satu Decision Tree tunggal.

Nilai Cohen’s Kappa pada kedua model juga tergolong rendah (0.086 dan 0.206), yang menurut interpretasi umum berada pada kategori slight agreement hingga fair agreement. Artinya, tingkat kesepakatan antara hasil prediksi model dengan nilai aktual masih jauh dari sempurna.

4.4 Faktor Penyebab Akurasi Rendah#

  1. Jumlah data terbatas relatif terhadap jumlah kelas — dataset hanya berisi 145 data yang harus diklasifikasikan ke dalam 8 kelas GRADE (0-7), sehingga rata-rata hanya sekitar 18 data per kelas. Setelah pembagian data uji (20%), jumlah data uji per kelas menjadi sangat sedikit untuk evaluasi yang representatif.

  2. Distribusi kelas tidak seimbang (class imbalance) — GRADE 1 memiliki 35 data sedangkan GRADE 4 hanya 10 data, menyebabkan model cenderung lebih akurat memprediksi kelas mayoritas dan kesulitan mengenali kelas minoritas.

  3. Jumlah kelas target yang banyak (8 kelas) — semakin banyak kelas target, semakin sulit bagi model untuk mencapai akurasi tinggi dibandingkan klasifikasi biner atau dengan sedikit kelas.

  4. Karakteristik fitur kuesioner — 30 fitur berskala ordinal (Likert) tidak selalu memiliki hubungan langsung dan linear terhadap nilai akhir mahasiswa, sehingga pola yang dipelajari model menjadi kurang kuat.

5. Kesimpulan#

  1. Proses data mining berhasil dilakukan menggunakan KNIME Analytics Platform melalui tahapan pembacaan data, penanganan missing value, seleksi fitur, transformasi tipe data, pembagian data latih/uji, hingga pemodelan dan evaluasi.

  2. Model Random Forest menunjukkan performa yang lebih baik dibandingkan Decision Tree, dengan akurasi 34.5% berbanding 23.1%, serta nilai Cohen’s Kappa yang juga lebih tinggi (0.206 berbanding 0.086).

  3. Akurasi kedua model tergolong rendah, yang lebih disebabkan oleh karakteristik dataset itu sendiri (jumlah data terbatas, jumlah kelas yang banyak, dan distribusi kelas tidak seimbang) dan bukan karena kesalahan pada alur workflow KNIME.

  4. Untuk penelitian lebih lanjut, disarankan dilakukan penambahan jumlah data, penanganan class imbalance (misalnya dengan teknik oversampling), atau penyederhanaan jumlah kelas target untuk meningkatkan performa model.
